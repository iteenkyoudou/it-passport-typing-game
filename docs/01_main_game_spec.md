# メインゲーム仕様書 (index.html)

## 1. 概要

ITパスポート試験の用語を学習するためのタイピングゲーム。穴埋め形式の問題に対して、3つの選択肢の中から正解をタイピングして入力する。モンスターとの戦闘演出（ドラクエ風）を取り入れ、ゲーム性を高めている。

---

## 2. 画面構成

### 2.1 イントロ画面 (`#intro-screen`)
- ページ読み込み時に全画面表示（z-index: 99999）
- 「画面をクリック または キーを押してスタート」と表示
- クリック/キー押下でストーリーテキストが1行ずつ順番に表示される（2.5秒間隔）
- ストーリーは14行（`INTRO_LINES`配列）で、ITパスポートの意義を語る導入演出
- SPACE キーまたはクリックでスキップ可能
- 終了時にフェードアウトしてメイン画面へ遷移
- イントロ専用BGM (`iwashiro_mikaduki_jelly.mp3`) がループ再生される

### 2.2 スタート画面 (`#start-screen`)
イントロ終了後に表示される。以下の要素を含む：

- **前回の記録表示** (`#my-best-card`): 最新のプレイ記録（連続正解数、獲得ゴールド、称号、日時）
- **モード選択** (`#mode-select`): 練習モード / 本気モードの2択
- **カテゴリ選択** (`#category-select`): すべて / ストラテジ系 / マネジメント系 / テクノロジ系 の4択（ラジオボタン）
- **サウンド設定** (`#sound-settings`): 全音ON/OFF、BGM ON/OFFボタン
- **スタートボタン** (`#start-btn`)
- **問題数情報** (`#q-count-info`): 選択中のカテゴリ・モードに応じた出題数を表示

### 2.3 ゲーム画面 (`#game-screen`)
3カラムレイアウト（左サイドバー / 中央ゲーム / 右サイドバー → ゲーム開始時に検索パネルに変わる）

#### 中央エリア（ゲーム本体）
- **HUD** (`#hud`): 問題番号 (X / Y)、獲得ゴールド表示
- **タイマー** (`#timer-text`, `#timer-bar-container`): 本気モード時のみ表示。残り秒数テキスト + プログレスバー
- **問題エリア** (`#question-area`): 穴埋め問題文を表示。空欄は `<span class="blank">` で下線表示
- **選択肢エリア** (`#choices-area`): 3つの選択肢をチップ形式で表示
- **タイピング入力エリア** (`#typing-area`): テキスト入力欄
- **ボスバトルUI** (`#boss-battle-ui`): ボス戦時のみ表示。ボス名 + HPバー
- **モンスター表示エリア** (`#monster-area`): モンスター画像、ダメージエフェクト、ゴールドドロップ演出
- **ステータス＆ナビゲーション** (`#status-nav-row`):
  - プレイヤーHPバー（本気モード時）
  - アイテムスロット（ポーション x2、腕時計 x2）
  - ナビゲーションボックス（ヒントテキスト）
- **フィードバック** (`#feedback`): 正解/不正解の結果表示
- **解説ボックス** (`#explanation-box`): 問題の解説を表示
- **次へボタン** (`#next-btn-area`): クリック or スペースキーで次の問題へ

#### 左サイドバー (`#sidebar-left`)
- **賞金ランキング** (`#ranking-card`): ゴールド獲得量順のランキングテーブル（Top10）

#### 右サイドバー (`#sidebar-right`)
- **称号解放者一覧** (`#title-list-card`): 各称号の解放状態と最初の解放者名

#### 検索サイドパネル (`#search-side-panel`)
ゲーム開始時にサイドバーの代わりに表示される。
- **検索チップ** (`#search-chips`): 選択肢をクリックしてGoogle検索
- **ブラウザバー** (`#search-browser-bar`): macOS風のドットボタン + URL表示
- **検索結果表示** (`#search-content`): iframe内にGoogleの検索結果を表示
- **フッター** (`#side-panel-footer`): 「Googleで開く」リンク
- 検索時に魔法エフェクト（パーティクルアニメーション）が発生

### 2.4 結果画面 (`#result-screen`)
#### 練習モード
- 正解数、不正解数、獲得ゴールド
- 正答率に応じたメッセージ（パーフェクト / すばらしい / いい調子 / まずまず / もう一度チャレンジ）

#### 本気モード

- 「もう一度挑戦する」ボタン

### 2.5 ゲームオーバー画面 (`#game-over-overlay`)
- 本気モードでHP=0になった時に表示

- 2秒後に自動で結果画面へ遷移

### 2.6 名前入力モーダル (`#name-input-overlay`)
- 本気モード終了時に表示
- プレイヤー名入力欄（最大12文字）
- 「登録する」「スキップ」ボタン
- 前回入力した名前がプリセットされる

### 2.7 称号獲得演出 (`#title-unlock-overlay`)
- ボス撃破時に全画面オーバーレイで表示（z-index: 9999）
- 「称号獲得！」テキスト + 称号名の豪華なアニメーション
- パーティクル（色付き丸30個 + 絵文字12個）が飛び散る

## 4. ゲームモード

### 4.1 練習モード (`normal`)
- 選択したカテゴリからランダムに **10問** 出題（`NORMAL_QUESTIONS = 10`）
- 制限時間なし
- HP・アイテムなし
- 全問終了後に結果画面を表示

### 4.2 本気モード (`mission`)
- 選択したカテゴリの **全問** をシャッフルして出題
- 1問あたり **45秒** の制限時間（`MISSION_TIME_PER_Q = 45`）
- プレイヤーHP制（最大100）
- アイテムシステムあり（ポーション、腕時計）
- HPが0になるとゲームオーバー
- 10問ごとにボスバトル発生
- ゲーム終了後に名前入力 → ランキング登録

## 5. タイピングゲームのロジック

### 5.1 入力方式
- テキスト入力欄 (`#typing-input`) にキーボードで入力
- `input` イベントでリアルタイム判定
- `Enter` キーでも判定実行
- 入力値は `normalize()` で正規化（小文字変換、空白除去、トリム）

### 5.2 選択肢の絞り込み
- 元の問題には複数の選択肢があるが、表示は **3つに絞る**（正解1 + 不正解2をランダム選択）
- 入力中に部分一致する選択肢チップがハイライトされる

### 5.3 正解判定
- 入力値が正解と完全一致した時点で即座に正解処理（Enter不要）
- 入力値が不正解の選択肢と完全一致した場合は不正解処理

### 5.4 正解時の処理 (`handleCorrect`)
1. タイマー停止
2. BGMの敗北カウント加算（3体ごとに曲切り替え）
3. 正解SE再生
4. スコア +100
5. 正解数カウント +1
6. 空欄に正解を表示
7. 選択肢チップを正解表示に変更
8. **ボス戦中の場合**: ボスダメージ加算 → HP=0なら撃破演出
9. **通常戦闘の場合**: モンスター撃破演出 → ゴールドドロップ → アイテムドロップ判定（本気モードのみ）
10. 解説表示
11. 「次へ」ボタン表示

### 5.5 不正解時の処理 (`handleWrong`)
1. タイマー停止
2. プレイヤーダメージ演出（画面赤点滅 + 画面揺れ）
3. 不正解SE再生
4. 不正解数カウント +1
5. 通常戦闘時はモンスターフェードアウト
6. 正解を表示
7. 選択肢チップに正解/不正解の色を付ける
8. 解説表示
9. **本気モード**: HPダメージ判定 → HP=0ならゲームオーバー
10. 「次へ」ボタン表示

### 5.6 タイムアウト時の処理 (`handleTimeout`)
- 不正解と同様の処理（HPダメージあり）
- モンスターはフェードアウト

### 5.7 スコア計算
- 正解1問につき **+100点** (`score += 100`)
- ゴールドは正解時にランダムで **100〜999G** 獲得（`Math.floor(Math.random() * 900) + 100`）
- ボス撃破時はゴールドドロップが **2回** 実行される

## 7. HPシステム（本気モードのみ）

### 7.1 基本パラメータ
| パラメータ | 値 |
|-----------|-----|
| 最大HP | 100 (`PLAYER_MAX_HP`) |
| 通常ダメージ | 1〜50 (`DMG_MIN`〜`DMG_MAX`) |
| クリティカルダメージ | 30〜99 (`CRIT_DMG_MIN`〜`CRIT_DMG_MAX`) |
| クリティカル発生率 | 10% (`CRIT_CHANCE = 0.1`) |

### 7.2 HPバー表示
- HP > 50%: 緑色
- HP 25〜50%: 黄色 (`hp-mid`)
- HP <= 25%: 赤色 (`hp-low`)

### 7.3 ダメージ演出
- **通常ダメージ**: 画面赤点滅 + 画面揺れ + ダメージ数値表示（3秒間）
- **クリティカルダメージ**: 上記に加えて「CRITICAL HIT!」テキスト + 激しい画面揺れ + 赤フラッシュ

## 8. アイテムシステム（本気モードのみ）

### 8.1 ポーション
- 最大所持数: **2個** (`MAX_POTION_STOCK = 2`)
- 回復量: **15〜30 HP**（ランダム）
- HP満タン時は使用不可
- アイテムスロットのクリックで使用
- 使用時: 画面に緑色の回復オーバーレイ + 「+X HP回復！」テキスト

### 8.2 腕時計
- 最大所持数: **2個** (`MAX_WATCH_STOCK = 2`)
- 効果: 制限時間を **45秒にリセット**
- アイテムスロットのクリックで使用
- 使用時: 金色フラッシュ + 時計回転エフェクト + 「時間リセット!」テキスト

### 8.3 アイテムドロップ
- 通常モンスター撃破時に判定（ボス戦ではドロップしない）
- ドロップ確率: **100%**（`ITEM_DROP_CHANCE = 1.0`）
- ポーションと腕時計が **50:50** の確率で決定
- どちらかが満タンの場合はもう一方がドロップ
- 両方満タンの場合はドロップなし

- **獲得ゴールド順**（多い順）で上位10件を保持

## 14. 検索機能

- パーティクル種類: spark（火花）、star（星）、ring（リング）
- パネル全体に光のグロー効果

## 16. データ保存

### 16.1 Firebase Firestore（クラウド）
- **プロジェクトID**: `it-passport-typing-game`
- **コレクション**: `game_data`
  - **ドキュメント `rankings`**: ランキングデータ（`records` 配列 + `updatedAt` タイムスタンプ）
  - **ドキュメント `unlocked_titles`**: 称号解放データ（`titles` オブジェクト + `updatedAt` タイムスタンプ）

### 16.2 localStorage（ローカルフォールバック）
| キー | 内容 |
|------|------|
| `itpassport_mission_records` | ランキングデータ（JSON配列、Top10） |
| `itpassport_player_name` | プレイヤー名 |
| `itpassport_unlocked_titles` | 称号解放データ（JSONオブジェクト） |

### 16.3 データフロー
1. ページ読み込み時: Firestoreからランキング・称号データを取得 → ローカルにもキャッシュ
2. ゲーム終了時: ローカルに保存 → Firestoreに同期
3. オフライン時: localStorageのデータをフォールバックとして使用
