# コースモード仕様書（course.html）

## 1. 概要

コースモード（`course.html`）は、ITパスポート試験の対策講座として機能する学習画面である。ドラゴンクエスト風のUI（黒背景・白枠・ゴールド色のテキスト）を採用し、RPGのステージ攻略のように問題を解き進める構成となっている。

ファイル: `/course.html`（単一HTMLファイルにCSS・JS・データを全て内包）

外部JSONデータ:
- `course/chapter1_stages1-4.json` -- ステージ1〜4のリッチデータ
- `course/chapter1_stages5-8.json` -- ステージ5〜8のリッチデータ
- `course/chapter1_stages9-11.json` -- ステージ9〜11のリッチデータ
- `course/chapter1_stage12_boss.json` -- ステージ12（ボス戦）のデータ

起動時にJSONファイルのフェッチを試み、失敗した場合はインラインデータにフォールバックする。

## 4. 学習フロー（通常ステージ）

各通常ステージは6ステップの学習フローで構成される。

```
Step 1: vocab      -- ことばを知ろう（用語学習）
Step 2: question   -- 問題に挑戦（メイン問題）
Step 3: knowledge  -- 知識を広げよう（関連知識）
Step 4: practice1  -- 練習問題 1
Step 5: practice2  -- 練習問題 2
Step 6: reward     -- カード獲得（ステージクリア報酬）
```

### 4.1 Step 1: ことばを知ろう（vocab）

- ステージテーマの紹介文をタイプライター演出で表示
- メイン問題に出てくる重要用語を1つずつカード形式で順次表示
- 各用語: 用語名、読み方（ふりがな）、説明文
- タップ/スペースキーで次のカードが表示される（順次表示システム）
- 最後に「覚えた！問題に進む」ボタンで次ステップへ

### 4.2 Step 2: 問題に挑戦（question）

メイン問題は実際のITパスポート試験の過去問を使用（出典表示あり）。

**表示順序（順次表示）:**

1. **問題文**（自動表示・タイプライター演出、モバイルでスティッキー）
2. **この問題は何を聞いている？**（問題文の噛み砕いた説明）
3. **ことばを復習する**（アコーディオン形式で用語一覧）
4. **選択肢**（4択、各選択肢にヒントボタン付き）-- ここで回答を待つ（data-wait）
5. **正解/不正解バナー + 全選択肢の解説**（回答後に表示）
6. **解き方の振り返り**（ステップ形式 + ひっかけポイント）
7. **理解度確認**（「次へ進む」or「もう一度読む」）

**回答フロー:**
1. 選択肢をタップ → ハイライト表示
2. 確認ダイアログ「この答えでいい？」→「決定！」or「選び直す」
3. 正解時: 正解バナー + EXP +20 のパーティクル演出
4. 不正解時: 不正解バナー + 所持ゴールドの50%を失う

**考え方のヒント:**
- アコーディオン形式で開閉可能
- 解法ステップをタイプライター演出で1つずつ表示
- 最後のステップ（正解直結）は非表示にし、回答後にフル表示

**各選択肢のヒント:**
- 選択肢ごとに「ヒント」ボタンがあり、押すとその選択肢が正解/不正解の理由をタイプライター演出で表示

### 4.3 Step 3: 知識を広げよう（knowledge）

- メイン問題に関連する知識を用語カード形式で順次表示
- 各カードは用語名 + 説明文（タイプライター演出）
- 最後に「覚えた！練習問題へ」ボタン

### 4.4 Step 4-5: 練習問題（practice1, practice2）

各ステージに2問の練習問題が用意されている。

- 構造はメイン問題と同様（問題文、ガイド、ことば復習、選択肢、解説）
- **選択肢のシャッフル**: ラベル（ア〜エ）は固定のまま、中身をランダムに入れ替え
- 正解時: EXP +15
- 不正解時: 所持ゴールドの30%を失う
- 回答後にタイプライター演出で解説を表示

### 4.5 Step 6: カード獲得（reward）

- ステージクリアの演出
- カード名、レアリティ（星表示）、サマリー、EXP・GOLD報酬を表示
- 初回クリア時のみ報酬が付与される
- `clearedStages` にステージIDを追加、`ownedCards` にカード名を追加
- ボス戦未クリアの場合、「ボス戦に備えてアイテムを買っておこう！」のTIPSを表示

## 5. ボスバトル仕様

### 5.1 ボス戦準備画面（screen-boss-prep）

- ボススプライト表示（ピクセルアート、アイドルアニメーション）
- 「ボス戦ではミスは許されない！」の警告メッセージ
- 所持ゴールド表示
- 持ち物表示（最大2個のアイテムスロット）
- ショップ: アイテム購入・売却が可能
- 「バトル開始！」ボタン → 確認モーダル → バトル開始

### 5.2 ショップアイテム

| アイテムID | 名前 | 価格 | 効果 |
|-----------|------|------|------|
| `life_stone` | 命のたま | 500G | 不正解でもリセットされない（自動使用、1回限り） |
| `sage_glasses` | 賢者のメガネ | 300G | 不正解の選択肢2つを消去して2択にする |

- 持ち物は最大2個まで
- 売却時は購入価格で全額返金

### 5.3 ボスバトル進行

**出題ソース:**
- 章内全ステージの `practiceQuestions`（全22問）からランダムに抽出
- 出題数: 3〜6問（ランダム）
- 問題はシャッフルされる
- `practiceQuestions` が空の場合は `BOSS_DATA.questions` にフォールバック

**バトルUI:**
- 2カラムヘッダー: ボス情報（名前・HPバー）+ ボススプライト
- HPバー: 正解するとHP減少アニメーション
- 問題カウンター: 「第N問 / 全M問」
- 問題文はタイプライター演出 → 完了後に選択肢表示

**回答フロー:**
1. 選択肢タップ → ハイライト
2. 確認ダイアログ「この答えでいい？」→「決定！」or「選び直す」
3. 正解: ダメージ演出（ヒットアニメーション + 画面揺れ）、EXP +15、GOLD +30〜49
4. 不正解:
   - 命のたま所持時: 自動消費して続行
   - 命のたま無し: 即敗北（全問正解が必須）

**賢者のメガネ使用:**
- ボス戦中に「賢者のメガネを使う」ボタンが表示される
- 使用すると不正解の選択肢2つが消去され2択になる
- 1回使い切り

**クリア条件:** 全問正解（途中で不正解 = 敗北、命のたまで1回だけ耐えられる）

### 5.4 ボスクリア報酬

- 称号「コンピュータの冒険者」を獲得
- ボーナスカード「メカドラゴン撃破の証」（レアリティ4）
- EXP +500、GOLD +1000
- `bossCleared` フラグを `true` に設定

### 5.5 ボス敗北時

- 「全問正解しないと撃破できない！」のメッセージ
- 「準備画面に戻る」「ステージ選択に戻る」の2ボタンを表示
- 何度でも再挑戦可能

## 6. レベル・経験値システム

### 6.1 レベル計算

```javascript
expForLevel(lv) = Math.floor(5 + lv * 0.35 + Math.pow(lv / 60, 2) * 3)
```

- 最大レベル: 999
- 累積EXPからレベルを逆算
- レベルアップ時: フラッシュ演出 + オーバーレイ表示

### 6.2 経験値獲得機会

| アクション | 獲得EXP |
|-----------|---------|
| タップ/クリックでコンテンツを進める | 2（ランダム報酬） |
| メイン問題を正解 | 20 |
| 練習問題を正解 | 15 |
| ボス戦で1問正解 | 15 |
| ステージクリア報酬 | 100〜150（ステージによる） |
| ボスクリア報酬 | 500 |

### 6.3 EXPバフシステム

- タップ報酬時に8%の確率で「EXP x1.2 GET!」バフ発動
- バフ発動後10回分のEXP獲得量が1.2倍になる
- バフ発動時: 画面フラッシュ + 中央バナー演出

### 6.4 ゴールド獲得・損失

| アクション | GOLD変動 |
|-----------|---------|
| タップ報酬（15%確率） | +1〜3 |
| ステージクリア | +200〜300 |
| ボスクリア | +1000 |
| ボス戦1問正解 | +30〜49 |
| メイン問題を不正解 | -50%（所持ゴールドの半分） |
| 練習問題を不正解 | -30%（所持ゴールドの30%） |

## 9. データ保存方法

### 9.1 LocalStorage キー

- キー: `it-course-state`
- 形式: JSON文字列

### 9.2 保存データ構造

```javascript
{
  exp: number,              // 累積経験値
  gold: number,             // 所持ゴールド
  level: number,            // 現在のレベル（EXPから自動計算）
  clearedStages: number[],  // クリア済みステージIDの配列
  ownedCards: string[],     // 獲得済みカード名の配列
  bossCleared: boolean,     // ボスクリアフラグ
  titles: string[],         // 獲得済み称号の配列
  currentTitle: string,     // 現在の称号名
  inventory: string[]       // 持ち物（アイテムIDの配列、最大2個）
}
```

### 9.3 初期値（デフォルト状態）

```javascript
{
  exp: 0,
  gold: 0,          // 注: テスト用コードでは gold:5000 に設定されている
  level: 1,
  clearedStages: [], // 注: テスト用コードでは [1,2,...,11] に設定されている
  ownedCards: [],
  bossCleared: false,
  titles: ['駆け出しの学び手'],
  currentTitle: '駆け出しの学び手',
  inventory: []
}
```

### 9.4 テスト用コードの注意点

現在のコードには以下のテスト用設定が含まれている（本番では修正が必要）:

1. `defaultState()` 内: `gold:5000`, `clearedStages:[1,2,...,11]` -- 全ステージ解放済み・ゴールド5000
2. `loadState()` 内: `localStorage.removeItem('it-course-state'); return defaultState();` -- 毎回リセットしてデフォルト状態を返す

### 9.5 保存タイミング

- `saveState()` が呼ばれるたびに LocalStorage に書き込み
- 呼び出し箇所: ステージクリア時、回答後のEXP/GOLD変動時、アイテム購入/売却時、ボスクリア時、レベルアップ時

## 10. 順次表示システム

コンテンツは「タップ/スペースキーで順次表示」される仕組みを採用。

### 10.1 仕組み

- `.reveal-block` クラスのdiv要素は初期状態で非表示（opacity:0, max-height:0）
- `.revealed` クラスが追加されると表示される（アニメーション付き）
- 最初のブロックは `revealed` クラスが初期から付与されている
- タップインジケーター「タップ or スペースキーで続きを読む ▼」が表示される

### 10.2 一時停止（data-wait）

- `data-wait="answer"` が設定されたブロックまで到達すると、順次表示が一時停止
- ユーザーが回答を完了すると `resumeReveal()` で再開
- 回答後、自動的に次のブロックが表示される

### 10.3 タイプライター演出

- `data-tw` 属性を持つ要素は、表示時にタイプライター演出（1文字ずつ表示）
- 速度: 30〜35ms/文字
- タイプ中に2文字ごとにタイピング音を再生
- スペースキーまたはタップでスキップ可能（即座に全文表示）

## 11. 演出・エフェクト

### 11.1 パーティクル演出

- **EXP獲得**: 青色の光パーティクル4個がクリック位置からサイドパネルのEXPエリアに飛ぶ
- **GOLD獲得**: ゴールド色のコインパーティクル3個がクリック位置からGOLD表示に飛ぶ
- **タップリップル**: タップ位置に青い波紋が広がるアニメーション

### 11.2 レベルアップ演出

1. 画面全体にゴールドのフラッシュ
2. 暗いオーバーレイ上に「LEVEL UP!」+ レベル数（拡大アニメーション）
3. タップで閉じる

### 11.3 称号獲得演出

1. 画面全体にゴールドのフラッシュ
2. 暗いオーバーレイ上に「新しい称号を手に入れた！」+ 称号名（拡大・発光アニメーション）
3. タップで閉じる

### 11.4 ボス戦演出

- **ボスアイドル**: 上下浮遊アニメーション（1.2秒周期）
- **ヒット**: 明滅 + 左右揺れアニメーション（0.5秒）
- **画面揺れ**: 正解時に画面全体が揺れる（0.4秒）

### 11.5 バフ獲得演出

1. 紫色の画面フラッシュ
2. 中央に「EXP x1.2 GET!」バナー（拡大→維持→フェードアウト、1.8秒）

## 14. JSONデータ読み込みフロー

```
1. ページ読み込み時、インラインのCOURSE_DATA・BOSS_DATAを初期値として使用
2. updateHeader() と renderStageGrid() を即座に実行
3. loadCourseDataFromJSON() を非同期で実行
   3a. course/chapter1_stages1-4.json, stages5-8.json, stages9-11.json を並列フェッチ
   3b. 成功: COURSE_DATA をJSONデータで上書き（ふりがな付きリッチデータ）
   3c. course/chapter1_stage12_boss.json をフェッチ
   3d. 成功: BOSS_DATA を上書き
   3e. renderStageGrid() を再実行
4. フェッチ失敗時: インラインデータをそのまま使用（コンソールに警告表示）
```

この仕組みにより、JSONファイルが利用できない環境でもインラインデータで動作する。
