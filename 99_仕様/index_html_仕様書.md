# index.html 仕様書
ITパスポート タイピングゲーム

---

## 概要
ITパスポート試験の用語を穴埋めタイピング形式で学習できるゲーム。
ドラクエ風のモンスターバトル演出で楽しく学べる。

## 画面構成

### 1. イントロ画面 (`#intro-screen`)
- ページ読み込み時に全画面表示（z-index: 99999）
- クリック/キー押下でストーリーテキストが1行ずつ順番に表示（2.5秒間隔）
- ストーリーは14行（`INTRO_LINES`配列）で、ITパスポートの意義を語る導入演出
- SPACE キーまたはクリックでスキップ可能
- 終了時にフェードアウトしてスタート画面へ遷移
- イントロ専用BGM (`iwashiro_mikaduki_jelly.mp3`) がループ再生

### 2. スタート画面 (`#start-screen`)
- **前回の記録表示** (`#my-best-card`): 最新のプレイ記録（連続正解数、獲得ゴールド、称号、日時）
- **モード選択** (`#mode-select`): 練習モード / 本気モードの2択
- **カテゴリ選択** (`#category-select`): すべて / ストラテジ系 / マネジメント系 / テクノロジ系（ラジオボタン）
- **サウンド設定** (`#sound-settings`): 全音ON/OFF、BGM ON/OFFボタン
- **スタートボタン** (`#start-btn`)
- **問題数情報** (`#q-count-info`): 選択中のカテゴリ・モードに応じた出題数を表示

### 3. ゲーム画面 (`#game-screen`)
3カラムレイアウト（左サイドバー / 中央ゲーム / 右サイドバー → ゲーム開始時に検索パネルに変わる）

#### 中央エリア（ゲーム本体）
- **HUD** (`#hud`): 問題番号 (X / Y)、獲得ゴールド表示
- **タイマー**（本気モードのみ）: 残り秒数テキスト + プログレスバー
- **問題エリア** (`#question-area`): 穴埋め問題文を表示。空欄は `<span class="blank">` で下線表示
- **選択肢エリア** (`#choices-area`): 3つの選択肢をチップ形式で表示
- **タイピング入力** (`#typing-area`): ローマ字 or 英語で入力
- **ボスバトルUI** (`#boss-battle-ui`): ボス戦・ラスボス戦時のみ表示。ボス名 + HPバー
- **モンスター表示エリア** (`#monster-area`): ドラクエ風モンスター画像、ダメージエフェクト、ゴールドドロップ演出
- **ステータス＆ナビゲーション** (`#status-nav-row`):
  - プレイヤーHPバー（本気モード時）
  - アイテムスロット（ポーション x2、腕時計 x2）
  - ナビゲーションボックス（操作ガイド表示）
- **フィードバック** (`#feedback`): 正解/不正解の結果表示
- **解説ボックス** (`#explanation-box`): 問題の解説を表示
- **次へボタン** (`#next-btn-area`): クリック or スペースキーで次の問題へ

#### 左サイドバー (`#sidebar-left`)
- **賞金ランキング** (`#ranking-card`): ゴールド獲得量順のランキングテーブル（Top10）

#### 右サイドバー (`#sidebar-right`)
- **称号解放者一覧** (`#title-list-card`): 各称号の解放状態と最新の解放者名

### 4. 検索サイドパネル (`#search-side-panel`)
ゲーム開始時にサイドバーの代わりに表示される。
- **検索チップ** (`#search-chips`): 選択肢をクリックしてGoogle検索
- **ブラウザバー** (`#search-browser-bar`): macOS風のドットボタン + URL表示
- **検索結果表示** (`#search-content`): iframe内にGoogleの検索結果を表示
- **フッター** (`#side-panel-footer`): 「Googleで開く」リンク
- 検索時に魔法エフェクト（パーティクル: spark、star、ring）が発生

### 5. 結果画面 (`#result-screen`)
#### 練習モード
- 正解数 / 不正解数 / 獲得ゴールド
- 正答率に応じたメッセージ（パーフェクト / すばらしい / いい調子 / まずまず / もう一度チャレンジ）
- 「もう一度遊ぶ」ボタン

#### 本気モード
- 正解数 / 全問数
- 獲得称号バッジ（ランクに応じた色・スタイル）
- 称号の説明文
- 正解、不正解、獲得ゴールドの統計
- 次の称号への案内メッセージ
- ランキングテーブル
- 「もう一度挑戦する」ボタン

### 6. ゲームオーバー画面 (`#game-over-overlay`)
- 本気モードでHP=0になった時に表示
- 「GAME OVER」タイトル + 敗因テキスト
- 2秒後に自動で結果画面へ遷移

### 7. 名前入力モーダル (`#name-input-overlay`)
- 本気モード終了時に表示
- プレイヤー名入力欄（最大12文字）
- 「登録する」「スキップ」ボタン
- 前回入力した名前がプリセットされる

### 8. 称号獲得演出 (`#title-unlock-overlay`)
- ボス撃破時に全画面オーバーレイで表示（z-index: 9999）
- 「称号獲得！」テキスト + 称号名の豪華なアニメーション
- パーティクル（色付き丸30個 + 絵文字12個）が飛び散る
- 3秒後に自動で閉じる
- ラスボス撃破時は虹色パーティクルの豪華版

## ゲームシステム

### ゲームモード

#### 練習モード (`normal`)
- 選択したカテゴリからランダムに **10問** 出題（`NORMAL_QUESTIONS = 10`）
- 制限時間なし
- HP・アイテムなし
- 全問終了後に結果画面を表示

#### 本気モード (`mission`)
- 選択したカテゴリの **全問** をシャッフルして出題
- 1問あたり **45秒** の制限時間（`MISSION_TIME_PER_Q = 45`）
- プレイヤーHP制（最大100）
- アイテムシステムあり（ポーション、腕時計）
- HPが0になるとゲームオーバー
- 10問ごとにボスバトル発生
- 全問クリア後にラスボス（ITかぐや）戦
- ゲーム終了後に名前入力 → ランキング登録

---

### タイピングゲームのロジック

#### 入力方式
- テキスト入力欄 (`#typing-input`) にキーボードで入力
- `input` イベントでリアルタイム判定
- `Enter` キーでも判定実行
- 入力値は `normalize()` で正規化（小文字変換、空白除去、トリム）

#### ローマ字入力対応
- `ROMAJI_MAP` に日本語→ローマ字の変換テーブルを定義（約200語）
- カタカナ・漢字の選択肢もローマ字入力で回答可能
- `normalizeRomaji()` で代替ローマ字を正規化:
  - ハイフン(-)を直前の母音の繰り返しに変換（例: `sa-` → `saa`）伸ばし棒対応
  - IME小文字入力の正規化（`uxo` → `wo`, `uxi` → `wi` 等）
  - 代替ローマ字（`sya` → `sha`, `si` → `shi`, `tu` → `tsu`, `hu` → `fu` 等）
  - 撥音正規化（`nn` → `n`）
  - 長音正規化（`ou` → `o`, `uu` → `u`）

#### 選択肢の絞り込み
- 元の問題には4つの選択肢があるが、表示は **3つに絞る**（正解1 + 不正解2をランダム選択）
- 入力中に部分一致する選択肢チップがハイライトされる

#### 正解判定
- 入力値が正解と完全一致した時点で即座に正解処理（Enter不要）
- ローマ字入力の場合も `matchesRomaji()` で一致判定
- 入力値が不正解の選択肢と完全一致した場合は不正解処理

#### 正解時の処理 (`handleCorrect`)
1. タイマー停止
2. BGMの敗北カウント加算（3体ごとに曲切り替え）
3. 正解SE再生
4. スコア +100
5. 正解数カウント +1
6. 空欄に正解を表示
7. 選択肢チップを正解表示に変更
8. **ボス戦中の場合**: ボスダメージ加算 → HP=0なら撃破演出
9. **通常戦闘の場合**: モンスター撃破演出 → ゴールドドロップ → アイテムドロップ判定（本気モードのみ）
10. 解説表示
11. 「次へ」ボタン表示

#### 不正解時の処理 (`handleWrong`)
1. タイマー停止
2. プレイヤーダメージ演出（画面赤点滅 + 画面揺れ）
3. 不正解SE再生
4. 不正解数カウント +1
5. 通常戦闘時はモンスターフェードアウト（ボス戦中は消えない）
6. 正解を表示
7. 選択肢チップに正解/不正解の色を付ける
8. 解説表示
9. **本気モード**: HPダメージ判定 → HP=0ならゲームオーバー
10. **1秒の待機後**に「次へ」ボタン表示（`setTimeout(() => showNextButton(), 1000)`）

#### タイムアウト時の処理 (`handleTimeout`)
- 不正解と同様の処理（HPダメージあり）
- モンスターはフェードアウト

### HPシステム（本気モードのみ）

| 定数 | 値 | 説明 |
|------|-----|------|
| `PLAYER_MAX_HP` | 100 | 最大HP |
| `DMG_MIN` | 1 | 通常ダメージ最小 |
| `DMG_MAX` | 50 | 通常ダメージ最大 |
| `CRIT_DMG_MIN` | 30 | クリティカルダメージ最小 |
| `CRIT_DMG_MAX` | 99 | クリティカルダメージ最大 |
| `CRIT_CHANCE` | 0.1 | クリティカル発生率（10%） |

- 不正解・タイムアウト時にダメージ
- HP 0でゲームオーバー

#### HPバー色変化
- HP > 50%: 緑色
- HP 25〜50%: 黄色 (`hp-mid`)
- HP <= 25%: 赤色 (`hp-low`)

#### ダメージ演出
- **通常ダメージ**: 画面赤点滅 + 画面揺れ + ダメージ数値表示（3秒間）
- **クリティカルダメージ**: 上記に加えて「CRITICAL HIT!」テキスト + 激しい画面揺れ + 赤フラッシュ

### アイテムシステム（本気モードのみ）

#### ポーション
| 定数 | 値 |
|------|-----|
| `MAX_POTION_STOCK` | 2 |
| `POTION_HEAL_MIN` | 15 |
| `POTION_HEAL_MAX` | 30 |

- クリックで使用
- HPを15〜30回復
- HP満タン時は使用不可
- 使用時: 画面に緑色の回復オーバーレイ + 「+X HP回復！」テキスト

#### 腕時計
| 定数 | 値 |
|------|-----|
| `MAX_WATCH_STOCK` | 2 |

- クリックで使用
- タイマーを45秒にリセット
- 使用時: 金色フラッシュ + 時計回転エフェクト + 「時間リセット!」テキスト

#### ドロップ確率
- 通常モンスター撃破時に100%の確率でアイテム判定（`ITEM_DROP_CHANCE = 1.0`）
- ポーションと腕時計が50:50の確率で決定
- どちらかが満タンの場合はもう一方がドロップ
- 両方満タンの場合はドロップなし
- ボス戦ではアイテムはドロップしない

### タイマーシステム（本気モードのみ）

#### 制限時間
- 1問あたり **45秒**（`MISSION_TIME_PER_Q = 45`）
- 0.1秒間隔で更新（`setInterval` 100ms）

#### 色変化
- 残り > 22秒: 緑色（通常）
- 残り 15〜22秒: 黄色 (`warning`)
- 残り <= 15秒: 赤色 (`danger`)

### ランキングシステム

#### ソート基準
- **獲得ゴールド順**（多い順）で上位10件を保持

#### 記録データ構造
```javascript
{
  streak: Number,    // 連続正解数（正解数）
  total: Number,     // 全問数
  category: String,  // カテゴリ
  date: String,      // ISO日時文字列
  name: String,      // プレイヤー名
  gold: Number,      // 獲得ゴールド
  wrongCount: Number, // 不正解数

```

#### 表示
- 左サイドバーにランキングテーブルを常時表示
- 本気モード結果画面にもランキングを表示
- 1位: 金色、2位: 銀色、3位: 銅色のハイライト
- 今回の記録は背景色付きでハイライト

## データ保存

### Firebase Firestore（クラウド）
- **プロジェクトID**: `it-passport-typing-game`
- **コレクション**: `game_data`
  - **ドキュメント `rankings`**: ランキングデータ（`records` 配列 + `updatedAt` タイムスタンプ）
  - **ドキュメント `unlocked_titles`**: 称号解放データ（`titles` オブジェクト + `updatedAt` タイムスタンプ）

### localStorage（ローカルフォールバック）
| キー | 内容 |
|------|------|
| `itpassport_mission_records` | ランキングデータ（JSON配列、Top10） |
| `itpassport_player_name` | プレイヤー名 |
| `itpassport_unlocked_titles` | 称号解放データ（JSONオブジェクト） |

### データフロー
1. ページ読み込み時: Firestoreからランキング・称号データを取得 → ローカルにもキャッシュ
2. ゲーム終了時: ローカルに保存 → Firestoreに同期
3. オフライン時: localStorageのデータをフォールバックとして使用
