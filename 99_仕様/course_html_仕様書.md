# course.html 仕様書
ITパスポート対策講座

---

## 概要
ITパスポート試験の過去問を使った学習型コースゲーム。
ドラゴンクエスト風のUI（黒背景・白枠・ゴールド色のテキスト）を採用し、ステージ制で進行する。
問題・解説・練習問題・カード獲得を通じて学習する。

ファイル: `course.html`（単一HTMLファイルにCSS・JS・データを全て内包）

## 学習フロー（通常ステージ）

各通常ステージは6ステップの学習フローで構成される。

```
Step 1: vocab      -- ことばを知ろう（用語学習）
Step 2: question   -- 問題に挑戦（メイン問題）
Step 3: knowledge  -- 知識を広げよう（関連知識）
Step 4: practice1  -- 練習問題 1
Step 5: practice2  -- 練習問題 2
Step 6: reward     -- カード獲得（ステージクリア報酬）
```

### Step 1: ことばを知ろう（vocab）
- ステージテーマの紹介文をタイプライター演出で表示
- メイン問題に出てくる重要用語を1つずつカード形式で順次表示
- 各用語: 用語名、読み方（ふりがな）、説明文
- タップ/スペースキーで次のカードが表示される
- 最後に「覚えた！問題に進む」ボタンで次ステップへ

### Step 2: 問題に挑戦（question）
メイン問題は実際のITパスポート試験の過去問を使用（出典表示あり）。

**表示順序（順次表示）:**
1. **問題文**（自動表示・タイプライター演出、モバイルでスティッキー）
2. **この問題は何を聞いている？**（問題文の噛み砕いた説明）
3. **ことばを復習する**（アコーディオン形式で用語一覧）
4. **選択肢**（4択、各選択肢にヒントボタン付き）-- ここで回答を待つ（data-wait）
5. **正解/不正解バナー + 全選択肢の解説**（回答後に表示）
6. **解き方の振り返り**（ステップ形式 + ひっかけポイント）
7. **理解度確認**（「次へ進む」or「もう一度読む」）

**回答フロー:**
1. 選択肢をタップ → ハイライト表示
2. 確認ダイアログ「この答えでいい？」→「決定！」or「選び直す」
3. 正解時: 正解バナー + EXP +20 のパーティクル演出
4. 不正解時: 不正解バナー + 所持ゴールドの50%を失う

### Step 3: 知識を広げよう（knowledge）
- メイン問題に関連する知識を用語カード形式で順次表示
- 各カードは用語名 + 説明文（タイプライター演出）
- 最後に「覚えた！練習問題へ」ボタン

### Step 4-5: 練習問題（practice1, practice2）
- 構造はメイン問題と同様（問題文、ガイド、ことば復習、選択肢、解説）
- 選択肢のシャッフル: ラベル（ア〜エ）は固定のまま、中身をランダムに入れ替え
- 正解時: EXP +15
- 不正解時: 所持ゴールドの30%を失う
- 回答後にタイプライター演出で解説を表示

### Step 6: カード獲得（reward）
- ステージクリアの演出
- カード名、レアリティ（星表示）、サマリー、EXP・GOLD報酬を表示
- 初回クリア時のみ報酬が付与される
- `clearedStages` にステージIDを追加、`ownedCards` にカード名を追加

## ボスバトル仕様

### ボス戦準備画面（screen-boss-prep）
- ボススプライト表示（ピクセルアート、アイドルアニメーション: 上下浮遊1.2秒周期）
- 「ボス戦ではミスは許されない！」の警告メッセージ
- 所持ゴールド表示
- 持ち物表示（最大2個のアイテムスロット）
- ショップ: アイテム購入・売却が可能
- 「バトル開始！」ボタン → 確認モーダル → バトル開始

### ショップアイテム

| アイテムID | 名前 | 価格 | 効果 |
|-----------|------|------|------|
| `life_stone` | 命のたま | 500G | 不正解でもリセットされない（自動使用、1回限り） |
| `sage_glasses` | 賢者のメガネ | 300G | 不正解の選択肢2つを消去して2択にする |
| `double_kendama` | 2倍のけん玉 | 400G | 次の正解が2倍ダメージになる（手動使用） |
| `time_watch` | シャッフルタイマー | 600G | 問題を別の問題に交換できる（手動使用） |

### 持ち物上限の動的拡張（`getMaxInventory()`）

| 条件 | 上限 |
|------|------|
| デフォルト | 最大2個 |
| 第2章ボスクリア後 | 最大3個 |
| 第4章ボスクリア後 | 最大4個 |

- 売却時は購入価格で全額返金

### ボスバトル進行

**出題ソース:**
- 現在章 + 過去全章の `practiceQuestions` を累積収集（`_chapterPqCache` でキャッシュ）
- 出題数: 章番号 + 2 問（第9章は12〜15問）
  - 例: 第1章ボス → 3問、第4章ボス → 6問、第8章ボス → 10問、第9章ボス → 12〜15問
- 問題はシャッフルされる
- `practiceQuestions` が空の場合は `BOSS_DATA.questions` にフォールバック

**バトルUI:**
- 2カラムヘッダー: ボス情報（名前・HPバー）+ ボススプライト
- HPバー: 正解するとHP減少アニメーション
- 問題カウンター: 「第N問 / 全M問」
- 問題文はタイプライター演出 → 完了後に選択肢表示

**回答フロー:**
1. 選択肢タップ → ハイライト
2. 確認ダイアログ「この答えでいい？」→「決定！」or「選び直す」
3. 正解: ダメージ演出（ヒットアニメーション + 画面揺れ）、EXP +15、GOLD +30〜49
4. 不正解:
   - 命のたま所持時: 自動消費して続行
   - 命のたま無し: 即敗北（全問正解が必須）

**賢者のメガネ使用:**
- ボス戦中に「賢者のメガネを使う」ボタンが表示される
- 使用すると不正解の選択肢2つが消去され2択になる
- 1回使い切り

**クリア条件:** 全問正解（途中で不正解 = 敗北、命のたまで1回だけ耐えられる）

### ボスクリア報酬
- 称号「コンピュータの冒険者」を獲得
- ボーナスカード「メカドラゴン撃破の証」（レアリティ4）
- EXP +500、GOLD +1000
- `bossCleared` フラグを `true` に設定

### ボス敗北時
- 「全問正解しないと撃破できない！」のメッセージ
- 「準備画面に戻る」「ステージ選択に戻る」の2ボタンを表示
- 何度でも再挑戦可能

## レベル・経験値システム

### レベル計算

```javascript
expForLevel(lv) = Math.floor(4 + 0.25 * lv + 0.002 * lv * lv)
```

- 最大レベル: 999
- 累積EXPからレベルを逆算
- レベルアップ時: フラッシュ演出 + オーバーレイ表示

### 経験値獲得機会

| アクション | 獲得EXP |
|-----------|---------|
| タップ/クリックでコンテンツを進める | 2（ランダム報酬） |
| メイン問題を正解 | 20 |
| 練習問題を正解 | 15 |
| ボス戦で1問正解 | 15 |
| ステージクリア報酬 | 100〜150（ステージによる） |
| ボスクリア報酬 | 500 |

### 章別EXP倍率（`CHAPTER_EXP_MULT`）

ステージクリア・問題正解などで得るEXPに章ごとの倍率が掛かる。

| 章 | 倍率 |
|----|------|
| 第1章 | ×0.5 |
| 第2章 | ×0.7 |
| 第3章 | ×0.9 |
| 第4章 | ×1.0 |
| 第5章 | ×1.2 |
| 第6章 | ×1.4 |
| 第7章 | ×1.6 |
| 第8章 | ×1.8 |

### EXPバフシステム
- タップ報酬時に8%の確率で「EXP x1.2 GET!」バフ発動
- バフ発動後10回分のEXP獲得量が1.2倍になる
- バフ発動時: 紫色の画面フラッシュ + 中央バナー演出（1.8秒）

### ゴールド獲得・損失

| アクション | GOLD変動 |
|-----------|---------|
| タップ報酬（15%確率） | +1〜3 |
| ステージクリア | +200〜300 |
| ボスクリア | +1000 |
| ボス戦1問正解 | +30〜49 |
| メイン問題を不正解 | -50%（所持ゴールドの半分） |
| 練習問題を不正解 | -30%（所持ゴールドの30%） |

## 模擬試験（第9章）

`id:9` の章。通常ステージなし・ボスなしの特殊モード。

### カテゴリ選択

| カテゴリ | 出題数 | 合格ライン |
|---------|--------|-----------|
| 全問題（総合） | 100問（tech:45 / strat:35 / mgmt:20） | 60問（60%） |
| テクノロジ系のみ | 45問 | 27問（60%） |
| ストラテジ系のみ | 35問 | 21問（60%） |
| マネジメント系のみ | 20問 | 12問（60%） |

カテゴリ比率定数: `MOCK_EXAM_RATIO = { tech: 45, strat: 35, mgmt: 20 }`

### 問題プールの構築
- 全8章のすべてのJSONファイルをfetchし、`mainQuestion` と `practiceQuestions` を収集
- カテゴリ（`テクノロジ系` / `ストラテジ系` / `マネジメント系`）で分類してシャッフル
- 選択カテゴリに応じて必要数を抽出

### タイマー
- 各問題に60秒タイマー（通常ステージ・ボス戦と同一仕様）

### 結果画面
- **合格/不合格** 表示（合格ライン: 正答率60%）
- スコア表示: `正解数 / 合計問題数（正答率%）`
- **カテゴリ別内訳**（総合モードのみ）: テクノロジ系・ストラテジ系・マネジメント系それぞれの正答数を表示
  - 60%以上: 緑色、60%未満: 赤色
- 「もう一度挑戦」「章の選択に戻る」ボタン

## データ保存

### アカウント管理

| localStorageキー | 内容 |
|-----------------|------|
| `course_accountName` | アカウント名 |
| `course_saveData_{アカウント名}` | セーブデータ（JSON） |

旧共通キー `course_saveData` からの一回限りマイグレーションに対応。

### Firebase Firestore（クラウド同期）
- **プロジェクトID**: `it-passport-typing-game`
- **コレクション**: `course_users`
  - ドキュメントID = アカウント名
  - セーブデータ + `createdAt` / `updatedAt` タイムスタンプ
- 保存はデバウンス制御（`_cloudSaveTimer`）
- デモモード時はFirestore操作をスキップ

### セーブデータ構造

## 順次表示システム

コンテンツは「タップ/スペースキーで順次表示」される仕組みを採用。

### 仕組み
- `.reveal-block` クラスのdiv要素は初期状態で非表示（opacity:0, max-height:0）
- `.revealed` クラスが追加されると表示される（アニメーション付き）
- 最初のブロックは `revealed` クラスが初期から付与
- タップインジケーター「タップ or スペースキーで続きを読む ▼」が表示

### 一時停止（data-wait）
- `data-wait="answer"` が設定されたブロックまで到達すると、順次表示が一時停止
- ユーザーが回答を完了すると `resumeReveal()` で再開

### タイプライター演出
- `data-tw` 属性を持つ要素は、表示時にタイプライター演出（1文字ずつ表示）
- 速度: 30〜35ms/文字
- タイプ中に2文字ごとにタイピング音を再生
- スペースキーまたはタップでスキップ可能（即座に全文表示）

## JSONデータ読み込みフロー

```
1. ページ読み込み時、インラインのCOURSE_DATA・BOSS_DATAを初期値として使用
2. updateHeader() と renderStageGrid() を即座に実行
3. loadCourseDataFromJSON() を非同期で実行
   3a. course/chapter1_stages1-4.json, stages5-8.json, stages9-11.json を並列フェッチ
   3b. 成功: COURSE_DATA をJSONデータで上書き（ふりがな付きリッチデータ）
   3c. course/chapter1_stage12_boss.json をフェッチ
   3d. 成功: BOSS_DATA を上書き
   3e. renderStageGrid() を再実行
4. フェッチ失敗時: インラインデータをそのまま使用（コンソールに警告表示）
```

## 中ボスシステム（新機能）

### 概要
